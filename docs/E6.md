# Dossier E6 – Laugimmo

## 1) Présentation

Laugimmo est une application web en PHP permettant de publier des annonces immobilières, de rechercher des biens, de réserver des dates et d’échanger des messages entre utilisateurs.

## 2) Contexte et objectifs

- Contexte : mise en situation d’une plateforme de réservation d’annonces immobilières.
- Objectifs : proposer un parcours complet (authentification, publication, recherche, réservation, échanges).
- Public cible : utilisateurs souhaitant publier/louer un bien, et administration du contenu.

## 3) Périmètre fonctionnel

- Utilisateur
  - Créer un compte, se connecter et se déconnecter
  - Consulter et filtrer les annonces
  - Créer/modifier/supprimer ses annonces et gérer les photos
  - Réserver une annonce sur une période disponible
  - Envoyer et consulter les messages liés à une annonce
  - Consulter ses réservations
- Admin
  - Accéder à une page d’administration
  - Supprimer des utilisateurs et des annonces

## 4) Architecture technique

- Stack
  - PHP + PDO (MySQL)
  - Routage “MVC léger” via [app/routes.php](../app/routes.php)
  - Contrôleurs : [app/Controllers/](../app/Controllers/)
  - Modèles / accès données : [app/Models/](../app/Models/)
  - Vues : [app/Views/](../app/Views/)
  - Tests unitaires : PHPUnit (dossier [TestUnitaire/](../TestUnitaire/))

- Points clés
  - Accès base via `get_pdo()` et variables d’environnement dans [config.php](../config.php)
  - Protection CSRF via `csrf_token()` / `verify_csrf()` dans [config.php](../config.php)
  - Gestion des sessions via `$_SESSION['user']` dans [config.php](../config.php)

## 5) Routes principales

- Public
  - `GET /` et `GET /annonces` : liste + filtres
  - `GET|POST /annonce/{id}` : détail + réservation + messagerie
  - `GET|POST /connexion` : connexion
  - `GET|POST /inscription` : inscription
  - `GET /deconnexion` : déconnexion
  - `GET /health` : état JSON (app + base)

- Authentifié
  - `GET|POST /creer-annonce` : création
  - `GET|POST /modifier-annonce/{id}` : édition
  - `GET /mes-annonces` : tableau de bord propriétaire
  - `GET /mes-reservations` : réservations du locataire
  - `GET /messages` : accès aux fils (liés aux annonces possédées ou réservées)

- Admin
  - `GET|POST /admin` : suppression utilisateurs / annonces

## 6) Base de données (résumé)

Schéma fourni pour correction : [../correction/schema.sql](../correction/schema.sql)

- `users`
  - Compte utilisateur + rôle admin
- `listings`
  - Annonces immobilières (titre, description, prix/nuit, pièces, localisation, catégorie, piscine)
- `listing_photos`
  - Photos associées à une annonce (chemin fichier)
- `reservations`
  - Réservations (dates, total, locataire, annonce)
- `messages`
  - Messages liés à une annonce (expéditeur, destinataire, contenu)

## 7) Sécurité (principes)

- Mots de passe
  - Stockage via `password_hash()` + vérification via `password_verify()` (service Auth)
- CSRF
  - Token CSRF en session et validation côté serveur
- Accès
  - Routes protégées par garde `auth` / `admin` dans [app/routes.php](../app/routes.php)
- Upload
  - Filtrage des types MIME autorisés (jpeg/png/webp) et renommage en fichier “safe”
